# FabricLab Community Roadmap

## Current direction

FabricLab is now positioned as a free public learning platform.

The short-term goal is:

- make every published chapter and lab openly accessible
- improve technical correctness continuously
- use practitioner feedback to harden labs and explanations

## What the platform already supports

- MDX-backed chapter content
- interactive lab simulator
- synced progress for signed-in users
- admin release controls for unpublished content
- Vercel deployment with Supabase-backed auth and metadata

## Near-term priorities

1. Readability and presentation polish across long chapters
2. Mobile compatibility and layout sanity checks
3. Technical-review sweeps chapter by chapter
4. Lab realism and command-surface refinement
5. Better contributor workflow around corrections and feedback

## Contribution themes we want

- chapter-level technical review
- screenshot-backed UI feedback
- lab sequencing feedback from working network engineers
- hardware/platform corrections with primary references
- reproducible bug reports for progress sync or release flow

## Support model

The current plan is:

- keep the learning platform free
- use lightweight optional support links
- defer any stronger monetization until the content and labs are mature enough to justify it

## Release philosophy

- content remains git-driven
- unfinished work can stay unpublished
- public access should favor openness over gating
- correctness matters more than shipping speed
