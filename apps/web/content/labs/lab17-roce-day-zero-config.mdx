## Scenario

The Spectrum-X leaf pair has passed the bring-up audit (Lab 16) â€” links are up, BGP sessions established, and the fabric is forwarding. Now the cluster owner has handed you the Day-0 RoCE requirements for the 32-GPU DGX H100 cluster attached to leaf-01 and leaf-02.

**Cluster owner requirements:**
- DSCP 26 trusted and mapped to TC3 (lossless) on all DGX-facing ports
- PFC enabled on priority 3 only (all other priorities: lossy)
- ECN min_threshold = 500 KB, max_threshold = 1.5 MB on TC3 egress
- CNP (DSCP 48) mapped to TC6, Strict Priority, PFC disabled
- MTU = 9216 bytes on all inter-node paths
- Performance acceptance: ib_write_bw >= 46 GB/s, ib_write_lat p99 <= 2us

You will apply the configuration using the NVUE shorthand on both leaf switches, verify each component of the QoS stack, then run RDMA performance benchmarks from dgx-node-01 to confirm the fabric is RoCE-ready.

**Devices:**

| Device | Type | Hardware | Prompt |

| leaf-01 | Leaf switch | SN5600 (Spectrum-4) | `leaf-01#` |
| leaf-02 | Leaf switch | SN5600 (Spectrum-4) | `leaf-02#` |
| dgx-node-01 | GPU server | DGX H100 (ConnectX-7) | `dgx-01:~$` |

## Task 1 â€” Apply RoCE Shorthand Configuration

</LabTerminal>

## Task 5 â€” Run ib_write_bw Bandwidth Test

From dgx-node-01, run ib_write_bw against dgx-node-02. Confirm >= 46 GB/s.

<LabTerminal nodeId="dgx-node-01" prompt="dgx-01:~$">
ib_write_bw -d mlx5_0 --iters 5000 --size 65536 192.168.100.2
</LabTerminal>

Expected pass output:
```
-------------------------------------------------------------------
 #bytes  #iterations  BW peak[GB/sec]  BW average[GB/sec]
 65536   5000           46.92            46.81
-------------------------------------------------------------------
```

If BW average shows ~12 GB/s: MTU mismatch. Check `ip link show eth0 | grep mtu` on both DGX nodes.

<LabHint level={1}>
If ib_write_bw shows significantly less than 46 GB/s, run `ibv_devinfo -d mlx5_0 | grep active_mtu`. If active_mtu is not 4096 (5), the Ethernet MTU is still at 1500. Fix: `ip link set eth0 mtu 9000` on both DGX nodes.
</LabHint>

<LabCheck taskId="bwVerified" passCriteria="ib_write_bw average >= 46 GB/s" />

## Debrief

You have completed a full RoCE Day-0 configuration sequence on a Spectrum-X leaf pair.

**Configuration order matters.** MTU must be set before QoS trust, and QoS trust before PFC/ECN. Applying ECN without DSCP trust means the lossless queue is never populated with RoCEv2 traffic.

**The shorthand vs manual trade-off.** The `qos roce` shorthand is sufficient for a homogeneous DGX-only cluster. Mixed workloads require the manual path to add DSCP 46 for NCCL and DSCP 10 for checkpointing.

**ib_write_bw is the acceptance gate.** 46.8 GB/s confirms: MTU correct, DSCP trust correct, lossless queue operating, no retransmissions.

**ECN is the primary mechanism; PFC is the safety net.** A fabric where PFC fires frequently (high `rx_pfc_xoff_frames`) is a fabric where ECN is misconfigured.
